About

Deployment of Deep learning models in the web

In this era of AI, Machine Learning and Deep Learning, machine computation and prediction is hugely beneficial to everyone in any field. The art of prediction is now better than ever. Human Computation is reduced significantly, whether it is face detection or stock market prediction. Almost Everyone wishes to use these powers. Creating models and predicting has never been easier.

However, there is one factor that is almost always overlooked while using these models for real life applications. Training deep neural networks, known as deep learning, is currently highly complex and computationally intensive. While GPUs have helped accelerate training, the amount of data as well as complexity of models have increased the computation need beyond the capability of a single GPU. For example, training on 2.5 million images on a single GPU can take 6 days on a simple model. A common user cannot enjoy these powers, when they have to wait for hours, and it might even require GPU, which may not be available in most PC. Also they are also unnecessarily exposed to the details of the underlying hardware and software infrastructure, including configuring expensive GPU machines, installing deep learning libraries, and managing the jobs during execution to handle failures and recovery. . Hence the part of deploying models is overlooked. Model Deployment takes the computational resources of the server rather than of the user. And if the server is backed, and connected via distributed computing, it can significantly increase its speed and uses server GPU, using zero to few, user's resources.

Some examples of usage of deep learning within application and services are: speech recognition, visual recognition, natural language understanding and classification, and language translation.

Deep Learning As a Service uses a distribution and orchestration layer that facilitates learning from a large amount of data in a reasonable amount of time across compute nodes. This project aims to take the deep learning models live, using Django Framework, and hosting the application, so that it is completely abstracted to the real world.
